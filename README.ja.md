# pixel-pet

`pixel-pet` は軽量なデスクトップペットのプロトタイプです。

コンセプト:

```txt
OSネイティブの常駐アプリ
+ かわいいピクセルアート
+ サイバーパンクな光
+ 穏やかなアイドルアニメーション
+ 短いセリフ
+ 将来的にはAIチャット、ミニゲーム、ペット交換を追加予定
```

このスターターは以下を使用しています:

```txt
Tauri v2
TypeScript
Vite
Canvas 2D
Rust製トレイメニュー
オリジナルの無料ピクセルスプライトシート
```

## ブラウザで試してみる

このペットは通常のブラウザでも動作します（ネイティブ機能はスムーズにフォールバックし、状態は`localStorage`に保存されます）。Rust/Tauriツールチェーンを使わずにローカルでビルド済みのコピーを試すには:

```bash
npm install
npm run build
npm run preview:web    # または: npx pixel-pet
```

`.github/workflows/pages.yml` ワークフローは、リポジトリでPagesが有効になっている場合に、このブラウザプレビューをGitHub Pagesに公開します。

## 実行方法

前提条件:

```txt
Node.js
Rust
お使いのOSに対応したTauri v2の前提条件
Linuxトレイビルド: AppIndicator/Ayatana AppIndicatorライブラリ
```

Arch/CachyOSでは、Linuxバンドルを作成する前に`libayatana-appindicator`をインストールしてください。Debian/Ubuntuでは、TauriのLinux前提条件に加えて`libayatana-appindicator3-dev`をインストールしてください。

インストールと実行:

```bash
npm install
npm run tauri:dev
```

ビルド:

```bash
npm run tauri:build
```

Windows、macOS、Linux向けのリリースビルドに関する注意事項は、`docs/RELEASE_BUILDS.md`に記載されています。
アップデート署名とマニフェストに関する注意事項は、`docs/UPDATER.md`に記載されています。

ローカルリリースチェック:

```bash
npm run check
npm run release:bundle
```

LinuxマシンでAppIndicatorシステムパッケージをインストールできない場合は、`npm run release:binary`を使用してインストーラバンドルなしでリリース実行ファイルを生成してください。

## 重要な注意点

- macOSの透明ウィンドウは、このプロトタイプでは`macOSPrivateApi: true`が必要です。
- これはローカルプロトタイプでは問題ありませんが、App Storeでの配布には適していません。
- 現在のペットはプロシージャルなCanvasアートです。最初の実際のペットアートが完成したら、スプライトシートに置き換えてください。
- このリポジトリは0BSDライセンスを使用しているため、コードや生成されたプレースホルダーアセットは広く再利用可能です。
- 古墳フレンズのアートは、実際のソースファイルが短いソースノートとともに追加され次第、`kofun-friend`ペットパック用に承認されます。
- アセットポリシーについては`docs/ASSETS.md`を参照してください。
- LLMは常時稼働しないように意図されています。AIはオンデマンドでのみ使用してください。
- 生成されたデスクトップバンドルは`src-tauri/target`以下に保存され、gitでは無視されます。リリースワークフローはそれらをアーティファクトとしてアップロードします。

## パフォーマンス予算

- アイドルおよび歩行アニメーションは10 FPSを目標としています。
- 反応アニメーションは12 FPSを目標としています。
- 睡眠アニメーションは4 FPSを目標としています。
- 非表示のドキュメントはキャンバス描画をスキップし、状態更新と保存のために5秒ごとのメンテナンスタックのみを実行します。
- 低干渉モードではアイドル時の動きをカットし、グリッチオーバーレイを抑制し、自発的な会話間隔を15分に引き上げます。
- フォーカスタイマーは15分、25分、45分の休憩リマインダー間隔をサポートし、完了した間隔に対して小さな気分/愛情の向上を報酬として与え、リマインダーをレート制限します。
- ボールゲームはアクティブな間のみアニメーション目標を20 FPSに引き上げ、その後は通常の低FPSペットスケジューラーに戻ります。
- フォーカスモードと低干渉モードはエネルギー消費を抑え、ユーザーが忙しい間は気分の低下を防ぎます。
- ブラウザプレビューで`?debugTiming=1`を追加するか、ローカルホストで実行して、開発用のタイミング読み出しを表示します。

## デスクトップシェルの注意点

- Tauriウィンドウはフレームレス、透明、固定サイズで、プラットフォームが`skipTaskbar`をサポートしている場合はタスクバーに表示されません。
- ペットウィンドウの上部ストリップをドラッグして位置を調整できます。
- `set`パネルでペットの表示、常に最前面、会話頻度、動作レベルを制御します。デスクトップの自動起動統合が追加されるまで、自動起動は無効として表示されます。
- トレイメニューには「Pixel Petを表示」「非表示」「終了」があります。
- ウィンドウを閉じるとトレイに非表示になります。アプリを終了するにはトレイメニューから「終了」を選択してください。
- macOSの透明性は`macOSPrivateApi`を使用しており、これはローカルプロトタイプでは許容されますが、App Storeでの配布には適していません。
- WindowsおよびLinuxのタスクバー非表示はデスクトップシェルに依存します。トレイメニューがフォールバックの操作インターフェースとして機能します。

## 状態の保存

デスクトップビルドでは、ペットの状態をTauriアプリデータディレクトリ内の`pet-state.v1.json`に保存します。このファイルには`schemaVersion`フィールドと`state`オブジェクトが含まれます。不正なJSONや未知のスキーマバージョンは無視され、アプリはデフォルトにフォールバックします。

一般的なローカルデバッグパス:

- macOS: `~/Library/Application Support/dev.local.pixel-pet/pet-state.v1.json`
- Windows: `%APPDATA%\dev.local.pixel-pet\pet-state.v1.json`
- Linux: `${XDG_DATA_HOME:-~/.local/share}/dev.local.pixel-pet/pet-state.v1.json`

既存のブラウザプレビューの`localStorage`状態は、初回の正常な読み込み時にデスクトップ状態ファイルに移行されます。通常のブラウザプレビューでは、Tauriコマンドが利用できないため、引き続き`localStorage`がフォールバックとして使用されます。

アクティブなペットIDは同じ状態ファイルに保存されます。各ペットは独自の気分、エネルギー、愛情の記憶を保持し、ウィンドウの位置と常駐アプリの設定はペットの切り替え間で共有されます。

低干渉モードも状態ファイルに保存されます。ウィンドウ内の`quiet`ボタンで切り替えることができ、ホスト統合は外部のフォーカスモードスイッチを反映するためにブラウザのカスタムイベントをディスパッチできます:

```js
window.dispatchEvent(new CustomEvent("pixel-pet:focus-mode", { detail: { enabled: true } }));
```

フォーカスタイマーの状態も同じファイルに保存されます。`focus`ボタンで現在のセッションを開始または一時停止し、間隔セレクターで休憩リマインダーの間隔を制御します。完了した間隔は気分と愛情を高め、繰り返しリマインダーを送信しません。

進行状況は、ペット状態内のバージョン管理された`progression.schemaVersion`オブジェクトに保存されます。XP、レベル、報酬アイテム、および報酬クールダウン履歴は、古い状態ファイルが安全にレベル1から始まるようにロード時に正規化されます。選択された各ペットは、気分、愛情、XP、レベル、および報酬インベントリを個別に保持します。

ダイアログは完全にオフラインで動作します。各ペットパックは、クリック、アイドル、睡眠、フォーカス、深夜、低エネルギーのトリガーに対する短いセリフを`manifest.json > personality.dialogue`で提供します。ランタイムはトリガーごとのクールダウンを適用し、44文字のバブル制限を遵守します。

## アクティビティイベント

ペットは実際に行っている作業に反応することができます。ホストプロセス（例: Claude Codeフック、エディタ拡張、またはgitラッパー）は、正規化されたアクティビティを報告するために`pixel-pet:activity`ブラウザカスタムイベントをディスパッチします:

```js
window.dispatchEvent(new CustomEvent("pixel-pet:activity", {
  detail: { kind: "done", source: "claude-code", label: "build" },
}));
```

`kind`は`start`、`working`、`waiting`、`done`、または`error`のいずれかです。`source`と`label`はオプションの短い文字列です。不明または不正なイベントは無視されます。

デスクトップではローカルファイルの受信ボックスも利用可能です。ホストは同じJSONオブジェクトを（1行ごとに）アプリデータディレクトリ内の`activity-inbox.jsonl`に追加し、アプリは低頻度でポーリングしてそれを処理します。すぐに使用できるClaude Codeフックは`tools/claude-code-activity-hook.mjs`にあります。ソースはエージェントに依存せず、`ActivitySource`（`src/pet/activitySource.ts`）としてモデル化されているため、git、エディタ、または非Claudeエージェントも同じ形状に適合します。詳細は`docs/ACTIVITY_HOOKS.md`を参照してください。

反応はアプリ全体の落ち着いた一貫性を保ちます: ペットは短い`react`を再生し、組み込みのセリフを表示し、気分を軽く刺激します。`done`イベントはクールダウン中に小さなアクティビティ報酬を与えますが、`error`はペナルティを与えません。各`kind`はレート制限されており、ホストがペットをスパムすることはできません。このパスはLLMを呼び出すことはなく、アニメーションループ上で実行されることもありません — 明示的なイベントにのみ反応し、常駐アプリの軽量性を保つ原則を反映しています。低干渉モードでは通常通りバブルが短縮されます。

## オプションのAIチャット

ペットはアイドルアニメーション、移動、フォーカスの促し、または環境ダイアログ中にLLMを呼び出すことはありません。ネットワーク接続されたアダプターは、ユーザーがチャットボックスにテキストを送信した後にのみ動作します。

プロバイダー:

- `rule`: オフラインのルールベースのフォールバック、ネットワーク不要。
- `ollama`: プロンプトと短いメモリ履歴を`http://localhost:11434/api/chat`に送信し、モデル`llama3.2`を使用。
- `openai`: プロンプトと短いメモリ履歴をOpenAI Responses APIに送信し、モデル`gpt-5.5`を使用。

OpenAIキーはブラウザのプロンプトで要求され、メモリ内にのみ保持されます。アプリの状態ファイルには書き込まれません。OpenAIの使用は、APIキーを所有するアカウントに費用が発生する場合があります。アダプターのタイムアウト、キーの欠如、Ollamaの利用不可、APIの失敗は、ペットの動作を壊すことなくルールベースのダイアログにフォールバックします。

## ボールゲーム

`ball`ボタンを押すと、透明なペットウィンドウ内で短いボール投げゲームが始まります。ゲームがアクティブな間にキャンバス内をクリックするとボールを投げることができ、ペットが追いかけてキャッチします。`stop`を押すか、ボールが止まるのを待つか、45秒経過すると通常のアイドル状態に戻ります。

報酬は意図的に小さく、オプションです: 明示的なインタラクションはクールダウン中に少量のXPを与え、フォーカス間隔は`focus-star`アイテムを付与し、ボールキャッチは`play-spark`アイテムを付与することがあります。ペットを放置してもXPの減少やペナルティはありません。

## ペットカプセル

`export`ボタンで選択したペットをローカルJSONカプセルとしてダウンロードできます。このスキーマには以下が含まれます:

```txt
schema
schemaVersion
compatibility.minAppVersion
compatibility.features
exportedAt
pet.id / pet.packId / pet.name / pet.seed
pet.traits
pet.ownerNote
pet.progression
```

`import`ボタンはローカルJSONカプセルを読み込み、スキーマバージョン、パックID、特性、進行状況を検証してから状態を変更します。不明なパック、不正なJSON、サポートされていないスキーマバージョン、無効な進行状況ペイロードは安全に失敗し、状態は変更されません。QR共有は意図的にこのJSONスキーマの上に追加されるフォローアップレイヤーとして残されています。

エクスポートされたカプセルには署名フィールドが予約されています。現在、インポートは有効なカプセルを未署名または未確認の署名済みとしてラベル付けしますが、どちらも信頼されたものとして扱われません。QRペイロードと署名計画については`docs/CAPSULE_SHARING.md`を参照してください。

## ペットパック

ペットパックは`assets/pets/<pack>/`以下に配置され、ビルド時にメタデータから読み込まれます。パックには以下が含まれている必要があります:

```txt
manifest.json
spritesheet.json
spritesheet.png
```

`manifest.json`はペットID、表示名、アセットファイル名、基本フレームサイズ、レンダースケール、期待されるアニメーションFPS/ループ設定を定義します。`spritesheet.json`はフレームサイズ、レイアウト行/列、アニメーション行、フレームリスト、FPS、ループ動作を定義します。無効または欠落したパックメタデータは無視され、レンダラーはデフォルトのパックまたはプロシージャルペットレンダラーにフォールバックします。

### コミュニティペットパック

バンドルされたパックに加えて、デスクトップアプリはランタイムでアプリデータディレクトリ内の`packs/`ディレクトリからパックを読み込みます。これにより、アプリを再ビルドすることなく誰でもペットを配布できます:

- macOS: `~/Library/Application Support/dev.local.pixel-pet/packs/<pack>/`
- Windows: `%APPDATA%\dev.local.pixel-pet\packs\<pack>\`
- Linux: `${XDG_DATA_HOME:-~/.local/share}/dev.local.pixel-pet/packs/<pack>/`

各`<pack>`フォルダには、バンドルされたパックと同じ`manifest.json`、`spritesheet.json`、`spritesheet.png`が含まれます。共有する前にフォルダを検証するには:

```bash
npm run assets:validate -- path/to/pack
```

外部パックもバンドルされたパックと同じ厳格な検証を受けます。不正なものはスキップされます。組み込みのパックIDが常に優先されるため、外部パックがバンドルされたものを上書きしたり偽装したりすることはできません。通常のブラウザプレビューにはパックディレクトリがなく、バンドルされたパックのみが表示されます。
