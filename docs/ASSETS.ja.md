# アセット

## 外部インスピレーションポリシー

`vscode-pets`、`virtual-puppet-project`、Kofun Friends、および類似のプロジェクトは、資産がこのリポジトリに明示的にリストされており、互換性のあるライセンスと許可の詳細がある場合を除き、インタラクションパターンの参考としてのみ使用されます。

第三者のキャラクターアート、スプライトシート、名前、サウンドエフェクト、UIスキン、またはブランド特有のビジュアル処理をこのリポジトリにコピーしないでください。ポケモンのようなアイデアは構造的である必要があります：ペットの収集、ケアループ、シンプルな移動、交換メカニクスは問題ありませんが、コピーされたキャラクター、シルエット、名前、タイプチャート、トレーナーコンセプト、またはフランチャイズのプレゼンテーションは許可されません。

## デフォルトのプレースホルダーアート

現在の`cyber-cat`、`cyber-penguin`、および`kofun-friend`スプライトシートは、このリポジトリのために生成されたオリジナルのピクセルアートです。これらはリポジトリの0BSDライセンスの下で公開されています。

次のコマンドで再生成できます：

```bash
npm run assets:generate
```

完全なアセットパイプラインを確認するには：

```bash
npm run assets:check
```

このコマンドはプレースホルダーシートを再生成し、すべてのペットパックを検証し、生成によってコミットされたファイルが変更された場合は失敗します。これにより、プレースホルダーアートが再現可能でありながら、壊れた手動作成のパックをキャッチできます。

## スプライト作成パイプライン

ペットパックは`assets/pets/<pack-slug>/`にあります。スラグは小文字のケバブケースにしてください。例えば、`cyber-cat`や`kofun-friend`です。

各パックには以下が含まれている必要があります：

```txt
manifest.json
spritesheet.json
spritesheet.png
```

オプションのソースファイル（例：`source.aseprite`）はこれらのファイルの横に置くことができますが、ランタイムは上記の3つのファイルのみを読み込みます。

### Asepriteエクスポート規約

単一の水平スプライトシートグリッドを使用してください：

```txt
row 0: idle
row 1: walk
row 2: sleep
row 3: react
```

サポートされているフレームサイズは`32x32`と`48x48`です。パック内のすべてのフレームは同じサイズを使用する必要があります。各行のフレームは左から右に配置し、`spritesheet.png`という名前のPNGをエクスポートします。

デフォルトのタイミング：

```txt
idle:  8 fps, ループ
walk: 10 fps, ループ
sleep: 4 fps, ループ
react: 12 fps, ループしない
```

`manifest.json`はペットID、表示名、フレームサイズ、スケール、アセットファイル名、期待されるFPS/ループ値、および`personality.dialogue`の下にある短いルールベースのダイアログラインを宣言します。`spritesheet.json`はPNGフレームサイズ、グリッドの行/列、アニメーション行、フレームインデックス、FPS、およびループフラグを宣言します。

4列4行の`48x48`パックの場合、PNGは`192x192`でなければなりません。`manifest.size.base`と`spritesheet.json.frame.width/height`の両方を`48`に設定してください。

新しいパックをコミットする前や生成されたアートを置き換える前に、次のコマンドを実行してください：

```bash
npm run assets:validate
npm run assets:check
```

バリデーションは、`manifest.json`の欠如、`spritesheet.png`の欠如、`spritesheet.json`の欠如、サポートされていないフレームサイズ、PNGの寸法不一致、アニメーション行の欠如、無効なフレームインデックス、マニフェスト/メタデータのFPS不一致、ダイアログトリガーの欠如、および44文字を超えるダイアログラインをキャッチします。

## Kofun Friends

プロジェクトの方向性：Kofun Friendsのアートは、実際のソースファイルが追加されるときに`kofun-friend`ペットのためにそのまま使用できます。

それまでは、コミットされた`kofun-friend`スプライトは生成されたプレースホルダーアートであり、コピーされたKofun Friendsのソースアートではありません。

## 第三者帰属テンプレート

生成されていない第三者アセットを追加する前に、アセットファイルの横にソースノートを保持してください：

```txt
source:
permission:
license:
copyright:
added_at:
notes:
```

ライセンスがこのリポジトリと互換性があり、再配布が許可されている場合にのみ、アセットをコミットしてください。これにより、将来のパッケージング、リリース、および帰属作業が簡単になります。
