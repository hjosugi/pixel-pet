# アーキテクチャ

## コア原則

常駐アプリを軽量に保つ。

```txt
アニメーション: ローカル状態マシン
対話: ルールベースの優先
LLM: オンデマンドのみ
ストレージ: ローカル優先
ネットワーク: オプションのみ
```

## ランタイム分割

```txt
Rust / Tauri
- ネイティブウィンドウ
- トレイメニュー
- 表示 / 非表示 / 終了
- 将来の自動起動
- 将来のローカルファイルストレージ

TypeScript / Canvas
- ペットアニメーション
- 状態マシン
- 対話バブル
- インタラクション
- オンデマンドチャットアダプターディスパッチ
- 将来のミニゲーム
```

## ペット状態マシン

```txt
アイドル
  -> 歩く   ランダムな低頻度の動き
  -> 眠る   低エネルギー
  -> 反応   クリック

歩く
  -> アイドル   数秒後
  -> 反応   クリック

眠る
  -> アイドル   エネルギーが回復したとき
  -> 反応   クリック
```

## AIプラン

アニメーションや基本的なペットの行動にLLMを使用しない。

AIは以下の目的のみに使用する：

```txt
- 短い会話
- 性格のバリエーション
- 日々の要約
- オプションの励まし
```

アダプター：

```txt
RuleBasedAiAdapter
OllamaAiAdapter
OpenAiAdapter
```

アダプターパスは明示的なユーザーアクションのみ：アイドルアニメーション、動き、環境対話、フォーカスの促しは決してLLMを呼び出さない。ブラウザは短いメッセージ履歴をメモリ内に保持し、タイムアウト、認証情報の欠如、ローカルOllamaの失敗、またはクラウドAPIの失敗時にはルールベースの対話にフォールバックする。

## アクティビティレイヤー

ホストは`pixel-pet:activity`カスタムイベントをディスパッチすることで、ペットが実際の作業に応答するようにできます。イベントは小さな種類のセット（`start`、`working`、`waiting`、`done`、`error`）に正規化され、種類ごとにスロットリングされ、既存の`react`状態にマッピングされ、少しのムード/報酬の促しが加わります。

```txt
生のホストペイロード
  -> normalizeActivityEvent   検証 + 制限
  -> reactToActivity          PetStateに対する純粋なリデューサー
  -> メインループが状態を適用し、話し、保存する
```

AIアダプターと同様に、これは観察のみ：LLMを呼び出すことはなく、アニメーションループ上で実行されることもなく、常駐アプリを軽量に保ちます。

## 外部ペットパック

バンドルされたパックはビルド時にコンパイルされます。デスクトップバックエンドは、アプリデータディレクトリ内の`packs/`フォルダーからコミュニティパックを追加で読み込み、生のマニフェスト、スプライトシートメタデータ、およびPNGバイトをフロントエンドに渡し、バンドルされたパックと同じ検証を実行します。組み込みIDが常に優先されるため、外部パックはバンドルされたものを上書きすることはできません。
